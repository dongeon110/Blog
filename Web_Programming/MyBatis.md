마이바티스 mybatis는 퍼시스턴스 프레임워크 중 하나입니다.  
먼저 퍼시스턴스 프레임워크에 대해 이해하고 마이바티스에 대해 살펴보겠습니다.  

# 퍼시스턴스 Persistence
퍼시스턴스는 영속성이라는 뜻으로 단어처럼 데이터의 지속성을 의미합니다.  
데이터를 생성한 프로그램이 **종료가 되더라도 사라지지 않고** 다시 실행하더라도 이전에 저장한 데이터를 다시 불러올 수 있습니다.  

퍼시스턴스 프레임워크는 데이터의 CRUD를 다루는 클래스 및 설정파일의 집합으로 mybatis도 그 중 하나입니다.


# 퍼시스턴스 프레임워크  
퍼시스턴스 프레임워크를 사용하면 JDBC 프로그래밍의 복잡함이나 번거로움 없이 간단한 작업만으로 데이터베이스와 연동되는 시스템을 빠르게 개발 할 수 있습니다.

퍼시스턴스 프레임워크에는   
SQL 문장으로 직접 DB 데이터를 다루는 **SQL 맵퍼**와  
자바 객체를 통해 간접적으로 DB 데이터를 다루는 **객체 관계 맵퍼**가 있습니다.

SQL맵퍼에는 대표적으로 mybatis가 있고  
객체 관계 맵퍼에는 하이버네이트(Hitbernate)와 탑링크(TopLink)가 있습니다.

# SQL맵퍼 vs 객체 관계 맵퍼
## 객체 관계 맵퍼의 장점?
객체 관계 맵퍼는 프레임워크에서 제공하는 API와 전용 객체 질의어를 사용하여 데이터를 다룹니다.  
프레임워크에서 제공하는 객체 질의어는 SQL보다 단순하여 배우기 쉽고 실행시에 DBMS에 맞추어 SQL문을 자동 생성하기 때문에 특정 DBMS에 종속되지 않는 애플리케이션을 만들 수 있습니다.  

## 객체 관계 맵퍼의 단점
객체 관계 맵퍼를 사용하려면 데이터베이스의 정규화가 잘 되어 있어야 합니다.  
그래야 테이블과 객체를 연결하기 쉽고, 객체를 통해 테이블의 데이터를 다루기가 쉽습니다.  
시스템을 개발하는 초기에는 데이터 모델링을 통해 정규화가 잘 되어 있지만, 유지보수 단계에 들어가면 잦은 기능 변경과 추가로 말미암아 초기에 잘 짜여진 데이터베이스의 구조가 흐트러지게 됩니다.  
이렇게 되면 이를 해결하기 위해 각종 편법을 동원하게 되고, 이 때문에 코드가 복잡해져서 오히려 개발 비용이 증가하게 됩니다.  

또 다른 단점으로, 데이터베이스의 특징에 맞추어 최적화를 할 수 없다는 것입니다.  
보통 데이터베이스들은 실행 성능을 높이고 데이터 처리를 쉽게 해주는 자신만의 특별한 기능을 가지고 있습니다.  
이런 기능을 활용할 수 있도록 데이터베이스 전용 SQL을 제공합니다.  
하지만, 객체 관계 맵퍼에서는 SQL문을 직접 작성하지 않기 때문에 이런 특장점을 활용 하기 어렵습니다.
