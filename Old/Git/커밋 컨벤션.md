# 커밋 컨벤션  


## 제목  
Feat: 새로운 기능 추가  
Fix: 버그 수정  
Docs: 문서 수정  
Style: 코드 포멧팅, 세미콜론 누락, 코드 변경 없는 경우  
Refactor: 리팩토링  
Test: 테스트 코드  

## 본문  
제목과 본문과 꼬릿말은 빈줄 한줄을 넣어서 구분한다.  


## 꼬릿말
꼬릿말은 ```유형: #이슈 번호``` 형식으로 사용  

Fixes: 이슈 수정중 (아직 해결 X)  
Resolves: 이슈를 해결 했을 경우  
Ref: 참고할 이슈  
Related to: 해당 커밋에 관련된 이슈 번호 (아직 해결 X)  

ex) Fixes: #45 Related to: #34, #23
