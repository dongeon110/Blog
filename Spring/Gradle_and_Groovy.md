# Gradle and Groovy  

# Gradle  
Gradle은 애플리케이션 빌드 자동화 도구 입니다.  
Gradle을 사용하면 프로그램 코드의 컴파일, 테스트, 빌드, 배치 등을 자동화할 수 있습니다.  
소프트웨어 패키지뿐만 아니라 정적인 웹 사이트 제작이나 문서 생성 프로젝트에도 사용할 수 있습니다.  

Gradle은 기존 빌드 도구들의 단점을 보강하고 장점을 잘 접목한 도구입니다.  
Ant로부터는 컴파일에서 파일의 이동 및 복사, 삭제, DB 스크립트의 실행, jar/war/ear/zip 파일 생성 등 다양한 작업에 적용할 수 있는 유연함과 강력한 기능들을 가져왔습니다.  
Maven으로부터는 강력한 의존 라이브러리 관리 기능을 가져왔습니다.  

이런 장점들 덕분에  
구글에서 무료로 배포하는 **안드로이드 스튜디오**는 프로젝트의 기본 빌드 도구로 Gradle을 사용하고 있고,  
**스프링 프레임워크** 도 빌드 시스템을 Gradle로 전환하였습니다.  


# Groovy  
Groovy는 Gradle의 빌드 스크립트를 작성할 때 사용되는 프로그래밍 언어이며, Python이나 Ruby, Perl과 비슷한 동적 프로그래밍 언어입니다.  
특히, Groovy DSL(Domain-Specific Language)은 빌드 스크립트를 작성하기에 XML보다 훨씬 유연한 구조를 제공합니다.  

Groovy는 스크립팅 언어의 특징을 갖고 있으면서도 컴파일 방식으로 동작합니다.  
즉, 인터프리터를 사용하여 Groovy소스를 바로 실행하는 것이 아니라, 컴파일을 먼저 한 다음 컴파일된 기계어 코드를 실행하는 것입니다.  

Groovy는 JVM에서 실행되기 때문에 자바와 연동이 쉽습니다.  
즉, 자바의 클래스를 직접 사용할 수 있습니다.  
자바 언어와 유사한 듯 하면서 훨씬 간결한 문법을 제공하기 떄문에 사용하기 쉽고 유지보수에 탁월합니다.  
자바 개발자라면 별 노력없이 Groovy를 사용할 수 있습니다.  
이것이 Gradle에서 Groovy를 도입한 이유입니다.  
자바 프로젝트를 수행하는 팀이라면, 개발자들이 이미 자바를 알고 있기 때문에 Groovy 언어를 빨리 배울 수 있습니다.
